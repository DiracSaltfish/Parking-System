# 停车场管理系统详细功能设计

## 1. 项目目标

本项目是一个双角色的停车场管理系统，采用前后端分离实现：

- 前端：`Vue 3 + Element Plus`
- 后端：`Spring Boot + Java`
- 数据存储：`Redis`

系统角色：

- 管理员
- 普通用户

当前目标是优先完成一条完整、稳定、可演示的业务主线。

## 5. 核心业务设计

### 5.1 统一登录入口

当前前端登录页采用单入口设计：

- 输入账号和密码
- 当账号为 `admin` 时，进入管理员登录逻辑
- 其他账号默认走普通用户登录逻辑
- 登录成功后根据 `role` 跳转页面

这样可以减少页面复杂度，也更适合课堂演示。

### 5.2 管理员登录

管理员使用账号密码登录后台。

登录后返回：

- 用户基本信息
- 角色信息
- Token

系统根据角色进入管理员端页面。

### 5.3 普通用户注册与登录

普通用户支持：

- 用户名注册
- 用户名密码登录

第一版不接短信验证，降低实现复杂度。

### 5.4 用户绑定车牌

普通用户可以绑定车辆。

当前支持：

- 查询绑定车辆
- 新增绑定车辆
- 修改绑定车辆
- 删除绑定车辆
- 指定主车牌

业务约束：

- 同一个车牌只能绑定一个用户
- 车牌格式需要通过前端与后端校验

### 5.5 管理员录入车辆入场

当前已实现。

流程如下：

1. 管理员输入车牌号
2. 可选输入车位号
3. 系统检查该车是否已经在场
4. 如果车牌已绑定用户，则自动关联用户
5. 如果未绑定用户，则按临时车辆处理
6. 系统生成停车记录
7. 以当前时刻作为入场时间开始计费

说明：

- 如果未填写车位号，系统会自动生成一个占位编号
- 录入完成后，当前停车列表会立即刷新

### 5.6 用户查询当前停车信息

普通用户登录后，可以查看自己当前车辆是否在停车场内。

如果车辆在场，会展示：

- 车牌号
- 当前车位
- 入场时间
- 已停车时长
- 当前应缴金额
- 支付状态

如果当前没有在场车辆，会返回“当前没有在场车辆”。

### 5.7 用户模拟缴费

当前已实现模拟支付流程，不接第三方支付平台。

流程如下：

1. 用户选择当前停车记录
2. 选择支付方式
3. 后端更新支付状态为已支付
4. 写入支付时间
5. 生成支付记录

这个方案适合课程项目演示，也能体现完整业务流程。

### 5.8 管理员查看全部车辆

管理员可以在“当前停车车辆”页面查看：

- 当前所有在场车辆
- 车牌号
- 车主名称或“临时车辆”
- 当前车位
- 入场时间
- 已停车时长
- 当前费用
- 支付状态

当前已支持：

- 按车牌号查询
- 当前列表实时刷新

### 5.9 停车记录查询

当前管理员和普通用户都可以查询停车记录：

- 用户只能查自己的记录
- 管理员可以查全部记录

可展示字段：

- 记录编号
- 车牌号
- 入场时间
- 出场时间
- 停车时长
- 原始费用
- 减免金额
- 最终费用
- 支付状态
- 记录状态

### 5.10 停车费豁免

这是系统中的扩展亮点功能。

当前设计已完成，但真实业务流程仍在下一阶段：

- 全额豁免
- 部分减免
- 原始费用与最终费用区分
- 记录操作人和操作时间

建议后续保留字段：

- `discountType`
- `discountAmount`
- `finalAmount`
- `exemptReason`
- `operatorId`
- `operateTime`

## 6. 当前已打通的演示主流程

推荐演示顺序：

1. 打开统一登录页
2. 管理员登录
3. 查看当前停车列表
4. 手动录入一辆新车入场
5. 展示系统从当前时刻开始计费
6. 退出后，普通用户登录
7. 查看绑定车辆
8. 查看当前停车信息
9. 查看停车记录和缴费记录
10. 完成模拟缴费
