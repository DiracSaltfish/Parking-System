# 停车场管理系统总体设计

## 1. 项目定位

本项目面向计算机实训课，目标是实现一个**前后端分离**的停车场管理系统，核心技术栈如下：

- 后端：`Java + Spring Boot`
- 前端：`Vue 3 + Element Plus`
- 数据存储：`Redis`
- 接口风格：`RESTful API`

系统以“单停车场管理”作为第一阶段目标，优先实现基础业务闭环：

1. 车辆入场
2. 车辆出场
3. 停车费用计算
4. 剩余车位统计
5. 停车记录查询
6. 管理员后台管理

这个范围适合课程实训，功能完整，开发难度可控，也便于后续扩展成“多停车场”“预约停车”“会员管理”等版本。

## 3. 总体架构设计

## 3.1 架构模式

采用前后端分离架构：

- 前端负责页面展示、表单交互、调用接口
- 后端负责业务逻辑、Redis 数据读写、权限校验
- Redis 负责缓存与业务数据存储

整体结构如下：

```text
前端 Vue/React
    |
    | HTTP / JSON
    v
Spring Boot 后端
    |
    | Spring Data Redis / Redisson
    v
Redis
```

### 3.2 后端分层建议

后端建议按经典分层设计：

- `controller`
  - 接收请求、返回响应
- `service`
  - 编写核心业务逻辑
- `repository` / `dao`
  - 封装 Redis 操作
- `entity` / `model`
  - 业务对象
- `dto` / `vo`
  - 接口参数和返回对象
- `config`
  - Redis、跨域、拦截器、安全配置

## 4. Redis 数据设计

> 说明：课程要求使用 Redis，可以实现；但 Redis 更适合高速读写和实时状态场景。  
> 如果后期要做复杂报表、强事务或长期归档，通常会再配合 MySQL。  
> 当前实训方案中，我们按“Redis 作为主存储”来设计，保证可以完整跑通。

### 4.1 核心对象

系统核心数据主要有：

- 管理员
- 车位
- 当前停车车辆
- 历史停车记录
- 收费规则
- 统计数据

### 4.2 Key 设计建议

#### 1. 管理员信息

```text
admin:{adminId} -> Hash
```

字段示例：

- `id`
- `username`
- `password`
- `name`
- `role`

#### 2. 车位信息

```text
parking:space:{spaceId} -> Hash
parking:space:all -> Set
parking:space:free -> Set
parking:space:occupied -> Set
```

车位 Hash 字段示例：

- `spaceId`
- `spaceCode`
- `type`
- `status`
- `floor`

说明：

- `parking:space:all` 保存全部车位 ID
- `parking:space:free` 保存空闲车位 ID
- `parking:space:occupied` 保存占用车位 ID

这样能快速统计车位状态。

#### 3. 当前在场车辆

```text
parking:current:{plateNumber} -> Hash
parking:current:plates -> Set
```

字段示例：

- `recordId`
- `plateNumber`
- `spaceId`
- `entryTime`
- `status`

说明：

- 当前在场车辆用 Hash 保存
- 所有在场车牌统一放到 Set 中，便于快速判断是否已入场

#### 4. 历史停车记录

```text
parking:record:{recordId} -> Hash
parking:record:list -> ZSet
parking:record:plate:{plateNumber} -> List
```

字段示例：

- `recordId`
- `plateNumber`
- `spaceId`
- `entryTime`
- `exitTime`
- `durationMinutes`
- `amount`
- `payStatus`

说明：

- `parking:record:list` 用 `ZSet`，score 可使用入场时间戳或出场时间戳，便于范围查询
- `parking:record:plate:{plateNumber}` 可保存该车牌的历史记录 ID 列表

#### 5. 收费规则

```text
parking:fee:rule -> Hash
```

字段示例：

- `freeMinutes`
- `pricePerHour`
- `dailyMaxAmount`

#### 6. 统计数据

```text
parking:stats:today -> Hash
```

字段示例：

- `entryCount`
- `exitCount`
- `income`

也可以按天扩展：

```text
parking:stats:day:20260310 -> Hash
```

## 5. 核心业务流程

### 5.1 车辆入场流程

1. 前端输入车牌号
2. 后端校验该车是否已在场
3. 从空闲车位集合中取一个车位
4. 生成停车记录 `recordId`
5. 写入当前停车信息
6. 更新车位状态为空闲 -> 占用
7. 更新统计数据

### 5.2 车辆出场流程

1. 根据车牌号查询当前停车信息
2. 获取入场时间并计算停车时长
3. 根据收费规则计算费用
4. 更新历史停车记录
5. 删除当前停车信息
6. 释放车位
7. 更新统计数据

### 5.3 费用计算规则

实训版建议采用简单规则：

- 前 `30` 分钟免费
- 超过后按小时收费
- 不满 1 小时按 1 小时计算

示例：

- 免费时长：`30` 分钟
- 每小时：`5` 元

计算方式：

```text
若停车时长 <= 30分钟，则费用为 0
否则费用 = ceil((停车时长 - 30) / 60) * 5
```

这样实现简单，也便于答辩讲解。

## 11. 实训项目落地建议

为了保证项目能做完，建议控制范围，不要一开始做太复杂：

- 先做单管理员登录，不做复杂权限
- 先做单停车场，不做多停车场联动
- 先做固定计费规则，不做复杂分时计费
- 先做人工入场/出场，不接车牌识别

这样能优先保证：

1. 功能闭环完整
2. 前后端分离明确
3. Redis 使用有实际场景
4. 项目答辩时逻辑清晰
