# 停车场管理系统总体设计

## 1. 项目定位

本项目面向计算机实训课，目标是实现一个**前后端分离**的停车场管理系统。

当前技术栈：

- 后端：`Java + Spring Boot`
- 前端：`Vue 3 + Element Plus`
- 数据存储：`Redis`
- 接口风格：`RESTful API`

系统采用“单停车场管理”作为当前版本范围，优先完成一个可演示、可答辩、可继续扩展的业务闭环。

---

## 2. 当前版本目标

当前版本重点不是做成复杂商业系统，而是完成以下主流程：

1. 登录认证
2. 用户绑定车牌
3. 管理员录入车辆入场
4. 系统按入场时间开始计费
5. 用户查询当前停车信息
6. 用户模拟缴费
7. 管理员查看当前车辆与历史记录

## 5. 整体架构设计

采用前后端分离架构：

```text
Vue 3 前端
   |
   | HTTP / JSON
   v
Spring Boot 后端
   |
   | Redis Repository
   v
Redis
```

### 5.1 前端职责

- 页面展示
- 路由控制
- 角色页面跳转
- 表单校验
- 调用后端 API

### 5.2 后端职责

- 登录认证
- 参数校验
- 业务逻辑处理
- 停车时长与费用计算
- Redis 数据存取
- 统一异常处理

### 5.3 Redis 职责

- 存储账号与会话信息
- 存储车辆绑定关系
- 存储当前停车记录
- 存储历史停车记录
- 存储支付记录

## 6. 后端分层设计

后端采用典型分层结构：

- `controller`
  - 接收请求，返回响应
- `service`
  - 处理业务逻辑
- `repository`
  - 封装 Redis 操作
- `dto`
  - 请求参数对象
- `vo`
  - 返回对象
- `config`
  - 配置项
- `common / exception / util`
  - 公共返回、异常处理、工具类

## 8. Redis 数据设计思路

本项目当前直接使用 Redis 作为主存储。

设计方式不是“表结构”，而是：

- 业务对象使用 `Hash`
- 关系索引使用 `Set / List`
- 主键序列使用 `String + INCR`
- Token 会话设置过期时间

当前已落地的主要 Key：

- `admin:{adminId}`
- `user:{userId}`
- `user:vehicle:{vehicleId}`
- `user:{userId}:vehicles`
- `parking:record:{recordId}`
- `parking:current:plate:{plateNumber}`
- `payment:{paymentId}`
- `auth:token:{token}`

## 9. 当前项目亮点

- 采用前后端分离，结构清晰
- 前端使用统一登录入口，按账号自动分流管理员与普通用户
- 管理员支持手动录入车牌入场
- 支持未绑定用户的临时车辆入场
- 停车费用按入场时间动态计算
- Redis 不只是缓存，而是参与业务数据建模
- 项目已经可以完整演示主流程
